# IRIS – Local Run Guide (no errors, fresh machine)

## 1) Prereqs
- Python 3.11+
- Node.js 18+
- Git
- (Optional but recommended) Virtualenv for Python

## 2) Clone
```
git clone <your-repo-url> iris
cd iris
```

## 3) Backend setup
```
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```
Create `.env` in repo root (minimal):
```
JWT_SECRET=changeme
OPENROUTER_API_KEY=
OPENROUTER_MODEL=meta-llama/llama-3-8b-instruct:free
admin_email=admin@iris.local
admin_password=ChangeMe123!
```

## 4) Pre-cache data (avoids yfinance hiccups)
```
python - <<'PY'
import yfinance as yf, pathlib
for t in ["AAPL","SPY"]:
    df = yf.download(t, start="2019-01-01", end="2024-12-31", progress=False, auto_adjust=True)
    p = pathlib.Path("app/data/cache") / f"{t}_2019-01-01_2024-12-31.parquet"
    p.parent.mkdir(parents=True, exist_ok=True)
    df.to_parquet(p)
    print("cached", t, len(df))
PY
```

## 5) Run backend
```
uvicorn app.main:app --reload --port 8000
```
Check: http://localhost:8000/health should return status ok.

## 6) Frontend setup
```
cd frontend
cp .env.example .env    # ensure VITE_API_BASE=http://localhost:8000
npm install
npm run dev
```
Open the URL shown (usually http://localhost:5173). Hard refresh if CSS looks old.

## 8) Run a sample strategy
Paste into Strategy Input:
```
Trade AAPL daily from 2019-01-01 to 2024-12-31. Go long when the 20-day SMA crosses above the 50-day SMA and RSI(14) is below 60. Exit when price closes below the 20-day SMA or RSI(14) exceeds 70. Risk no more than 2% of capital per trade; cap position at 50% of equity. Use 10 bps commission and 5 bps slippage.
```

Then RUN IRIS. Equity/metrics should render; errors will show with detail if data fetch fails.

## 9) Common fixes
- If charts look empty: hard refresh (Ctrl+Shift+R). Ensure backend logs show data loaded (no yfinance errors). Stooq fallback requires internet.
- If auth fails: verify backend is on port 8000 and frontend VITE_API_BASE matches.
