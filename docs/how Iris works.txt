Project Name:
IRIS: Intelligent Reasoning & Inferential Simulator.


Problem Statement:
Agentic Al Backtesting Tool
Traders want to test strategies before risking real money - but today's tools demand either coding expertise or expensive platforms. There's no quick way to describe a strategy in plain English and immediately see how it would have performed. build an agent that accepts a trading strategy written in plain English, parses it into executable rules using an LLM, runs a historical backtest with realistic friction modelling, and returns a performance tearsheet - equity curve, Sharpe ratio, drawdown - within seconds.


Expected Outcomes:
An Al-powered backtesting tool where a user types a strategy in natural language "buy when the 50-day MA crosses above the
200-day MA, sell when RSI exceeds 70" and an Al agent automatically parses the intent, runs a realistic historical simulation, and returns a clean performance tearsheet: equity curve, Sharpe ratio, max drawdown, and win rate.
The core agentic loop is simple: the LLM translates natural language into structured trading rules, the backtest engine executes them against historical data, and results feed back so the user can iterate conversationally.

Approach: 
The trader will give the input prompt in English. This prompt includes all the requirements, the trader wants to test.
The prompt is an input to "Manager Agent"

1) Manager Agent
This agent does the NLP of the prompt and understands what the trader's strategy is. 
It then instructs the "Trader Strategy Agent" to do EXACTLY what the trader wants to do.
The Manager then also, allocates another NON-trader strategy agent do to the exact same thing but this agent (Expert Agent) already has set of algorithms to follow.
thus, manager- understand, allocates task to 2 agents
the solution also goes through the manager to the trader in the end

the manager is also connected to the "Automator Agent"

2) Trader Strategy Agent
This agent carries out exactly what the manager as instructed it to do ie the traders strategy
it scraps out the idea and does the simulation
the result is given to "Verifier Agent"

3) Expert Agent:
3.1]  Risk Analysis
This agent is responsible for analyzing the risk
it is called if the trader wants to analysis his risk strategy, thus this acts as a comparator
it uses Monte-carlo algorithm and GARCH

3.2] Derivatives & Pricing
This agent is responsible for accounting for the profit on derivatives wrt price
it is called if the trader wants to analysis his derivative strategy, thus this acts as a comparator
it uses black-scholes and binomial tree

3.3] Portfolio Constructor
This is responsible to build a good portfolio
it is called if the trader wants to analysis his portfolio strategy, thus this acts as a comparator
it uses Mean-Variance, Black-Litterman

3.4] Alpha Generation & Signal Research Agent
Alpha Generation refers to investment strategies designed to outperform a benchmark (generating "excess return" or alpha) through superior stock selection, timing, or, increasingly, quantitative modeling. Signal Research involves identifying predictive, data-driven metrics—such as sentiment analysis, alternative data, or price patterns—to fuel these strategies.
it is called if the trader wants to analysis his Alpha & singnal strategy, thus this acts as a comparator.
it uses Kalman Filter, Pairs Trading

3.5] Fixed Income & Rates Agent
Fixed income refers to investment securities that pay investors regular, predetermined interest or dividends until a specified maturity date, when the principal is returned. Common examples include government/corporate bonds and Certificates of Deposit (CDs). These instruments are generally lower-risk than equities, acting as a stable, predictable income stream for portfolios.
it is called if the trader wants to analysis his Fixed income strategy, thus this acts as a comparator. 
it uses 
a) duration & convexity analysis, 
b) short rate models

3.6] Execution & Market Microstructure Agent
Market microstructure studies how financial assets are traded, focusing on the mechanics of price formation, order execution, and liquidity. It examines rules, trading systems (e.g., Bookmap, QuantStart), and behavior. Execution is the process of completing buy/sell orders, balancing speed, cost, and price impact, as explained in this StoneX guide.
it is called if the trader wants to analysis his Microstructure strategy, thus this acts as a comparator.
it uses HMM, VWAP


4) Verifier Agent
this input 2 strategies:
one is from trader strategy agent, another is from one of the expert agent
it also has the initial requirements that was set by the manager agent
this agent just makes sure that the work assigned by the manager is carried out exactly as expected.
if not, the verifier tell the respected agent (trader strategy agent or expert agent) to redo the task
if the simulation is correct, it passes the results to comaparator


5) Comparator Agent
this compares the 2 simulation (trader strategy agent & expert agent) and returns a performance tearsheet - equity curve, Sharpe ratio, drawdown to the manager


at the end the manager explain the tearsheet to the trader and ask if he want to automate his strategy or the expert's strategy

accordingly, manager calls the "Automator Agent"


6) Automator Agent
this automates the process of implementing the strategy as told by manager agent
once done it flags the completion
or the error if the strategy was not implemented to the manager

the manager sends this flag in English to the trader
